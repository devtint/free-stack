Setup & Config

Do this once per machine. Git needs to know who you are before it can make commits.

First-time setup

terminal
# Set your identity (used in all commits) $ git config --global user.name "Tint Naing Win" $ git config --global user.email "188316439+devtint@users.noreply.github.com" # Set default branch name to main (not master) $ git config --global init.defaultBranch main # Set VS Code as default editor $ git config --global core.editor "code --wait" # View your config $ git config --list user.name=Tint Naing Win user.email=188316439+devtint@users.noreply.github.com init.defaultbranch=main

SSH key setup (recommended over HTTPS)

terminal
# Generate SSH key $ ssh-keygen -t ed25519 -C "188316439+devtint@users.noreply.github.com" Generating public/private ed25519 key pair. Enter file in which to save the key (~/.ssh/id_ed25519): [press Enter] Enter passphrase (empty for no passphrase): [press Enter] # Copy public key to clipboard $ cat ~/.ssh/id_ed25519.pub ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... 188316439+devtint@users.noreply.github.com # → Paste this in GitHub: Settings → SSH Keys → New SSH Key # Test SSH connection $ ssh -T git@github.com Hi devtint! You've successfully authenticated, but GitHub does not provide shell access.

Check version & aliases

terminal
$ git --version git version 2.47.1 # Useful aliases — add to ~/.gitconfig $ git config --global alias.st status $ git config --global alias.co checkout $ git config --global alias.br branch $ git config --global alias.lg "log --oneline --graph --decorate --all" # Now you can use: git st, git co, git br, git lg
02

Basic Workflow

The commands you use every single day. Master these first.

Start a project

terminal
# Option A: Start fresh locally — create a new folder and init git $ mkdir my-telegram-bot $ cd my-telegram-bot $ git init Initialized empty Git repository in ~/my-telegram-bot/.git/ # Option B: Clone an existing repo (SSH) $ git clone git@github.com:devtint/TempMail.git Cloning into 'TempMail'... remote: Enumerating objects: 47, done. # Clone into a specific folder name $ git clone git@github.com:devtint/TempMail.git my-project # Clone a specific branch only $ git clone --branch dev git@github.com:devtint/TempMail.git

The daily commit cycle

terminal — everyday flow
# 1. See what changed $ git status On branch main Changes not staged for commit: modified: bot.py Untracked files: .env.example # 2. Stage specific files $ git add bot.py $ git add .env.example # Stage ALL changed + new files at once $ git add . # Stage only modified files (not untracked) $ git add -u # Interactively stage chunks of a file $ git add -p bot.py # 3. Commit with a message $ git commit -m "feat: add /start command with welcome message" [main a3f12c8] feat: add /start command with welcome message 1 file changed, 15 insertions(+), 2 deletions(-) # Shortcut: stage all modified + commit in one line $ git commit -am "fix: handle empty message gracefully"

Push & pull

terminal
# Push to remote (origin = GitHub) $ git push origin main Enumerating objects: 5, done. To github.com:devtint/TempMail.git c5f4ee4..a3f12c8 main -> main # First push of a new branch (set upstream) $ git push -u origin feature/webhook # Pull latest changes from remote $ git pull origin main # Pull with rebase instead of merge (cleaner history) $ git pull --rebase origin main # Fetch without merging (inspect first) $ git fetch origin $ git diff main origin/main

Connect local repo to GitHub

terminal — new project setup
# After creating repo on github.com $ git remote add origin git@github.com:devtint/my-bot.git $ git branch -M main $ git push -u origin main # Check remotes $ git remote -v origin git@github.com:devtint/my-bot.git (fetch) origin git@github.com:devtint/my-bot.git (push) # Change remote URL (HTTPS → SSH) $ git remote set-url origin git@github.com:devtint/my-bot.git

03

Branching, Merge & Rebase

Branches let you work on features without breaking main. Use them even as a solo builder — one branch per feature or fix.

Create & switch branches

terminal
# Create a new branch and switch to it (modern way) $ git switch -c feature/tiktok-downloader Switched to a new branch 'feature/tiktok-downloader' # Old way (same result) $ git checkout -b feature/tiktok-downloader # Switch to existing branch $ git switch main # List all branches $ git branch * feature/tiktok-downloader main # List remote branches too $ git branch -a # Delete a branch (after merging) $ git branch -d feature/tiktok-downloader # Force delete (even if not merged) $ git branch -D feature/old-test # Delete remote branch $ git push origin --delete feature/tiktok-downloader

Merge (preserves branch history)

terminal — merge feature into main
# 1. Switch to main first $ git switch main # 2. Merge feature branch in $ git merge feature/tiktok-downloader Updating c5f4ee4..a3f12c8 Fast-forward # No-fast-forward merge: always creates a merge commit $ git merge --no-ff feature/tiktok-downloader # If there's a conflict: CONFLICT (content): Merge conflict in bot.py # → Edit the file to resolve conflicts, then: $ git add bot.py $ git commit -m "merge: resolve conflict in bot.py"

Rebase (linear, clean history)

terminal — rebase onto main
# Rebase your feature branch onto latest main $ git switch feature/warp-gen $ git rebase main Successfully rebased and updated refs/heads/feature/warp-gen. # Interactive rebase: squash/edit/reorder last 3 commits $ git rebase -i HEAD~3 # → Opens editor. Change 'pick' to 'squash' to combine commits. # Abort a rebase if things go wrong $ git rebase --abort
💡

Merge vs. Rebase: Use merge for shared/public branches (preserves history). Use rebase on your own local feature branches before merging (cleaner linear history). Never rebase a branch others are working on.

04

Undo & Fix

Git lets you undo almost anything. Know which tool to use and when.

✏️

Amend

Edit the last commit — change the message or add a forgotten file. Only if not pushed yet.

📦

Stash

Temporarily shelve changes without committing. Switch branches with a clean state.

⏪

Reset

Move HEAD backwards. Unstage files or throw away commits entirely.

↩️

Revert

Create a new commit that undoes a previous one. Safe on public branches.

🍒

Cherry-pick

Apply a specific commit from another branch to the current branch.

🔍

Restore

Discard changes in a file, or unstage a file without losing changes.

Amend — fix the last commit

terminal
# Fix typo in last commit message $ git commit --amend -m "feat: add /start command with welcome message" # Add a forgotten file to last commit (keeps same message) $ git add forgotten_file.py $ git commit --amend --no-edit # ⚠ If already pushed, force push is required $ git push --force-with-lease origin main

Stash — save work temporarily

terminal
# Stash current changes (named stash) $ git stash push -m "WIP: webhook handler refactor" Saved working directory and index state On main: WIP: webhook handler refactor # Stash including untracked files $ git stash push -u -m "WIP: new feature" # List all stashes $ git stash list stash@{0}: On main: WIP: webhook handler refactor stash@{1}: On main: quick fix test # Apply latest stash (keeps stash in list) $ git stash apply # Apply specific stash $ git stash apply stash@{1} # Apply and remove from list $ git stash pop # Drop a specific stash $ git stash drop stash@{0} # Clear all stashes $ git stash clear

Reset — move or unstage

terminal
# Unstage a file (keeps changes in working dir) $ git reset HEAD bot.py # Modern equivalent: $ git restore --staged bot.py # Undo last commit — keep changes staged $ git reset --soft HEAD~1 # Undo last commit — keep changes unstaged $ git reset --mixed HEAD~1 # default if no flag # ⚠ DESTRUCTIVE: Undo last commit + delete all changes $ git reset --hard HEAD~1 # ⚠ DESTRUCTIVE: Discard ALL uncommitted changes in a file $ git restore bot.py # Go back to a specific commit $ git reset --hard a3f12c8
🚨

git reset --hard deletes uncommitted work permanently. If you're not sure, use git stash first to save your changes, then reset. You can also use git reflog to recover lost commits.

Revert — safe undo on public branches

terminal
# Revert a specific commit (creates a new "undo" commit) $ git revert a3f12c8 [main d9c44f1] Revert "feat: add /start command" # Revert last commit without opening editor $ git revert --no-edit HEAD # Revert a range of commits $ git revert HEAD~3..HEAD

Cherry-pick — grab one commit from another branch

terminal
# Apply commit a3f12c8 from any branch to current branch $ git cherry-pick a3f12c8 [main d3e1a2b] feat: add /start command with welcome message # Cherry-pick without auto-committing (stage only) $ git cherry-pick --no-commit a3f12c8 # Cherry-pick multiple commits $ git cherry-pick a3f12c8 b7e9d4f # Abort cherry-pick if conflict $ git cherry-pick --abort

Reflog — recover "lost" commits

terminal — git's safety net
# Show every HEAD movement (even after reset --hard) $ git reflog d3e1a2b (HEAD -> main) HEAD@{0}: commit: feat: add /start a3f12c8 HEAD@{1}: reset: moving to HEAD~1 c5f4ee4 HEAD@{2}: commit: fix: webhook handler # Recover a "lost" commit by checking out its hash $ git checkout a3f12c8 # Or restore to that point $ git reset --hard HEAD@{1}

History & Inspection

Read the repo's history, find bugs, compare changes, and understand what changed and why.

git log — view commit history

terminal
# Full log $ git log commit a3f12c8... Author: Tint Naing Win <...> Date: Mon Apr 22 00:30:00 2026 feat: add /start command with welcome message # One-line format (most useful) $ git log --oneline a3f12c8 feat: add /start command with welcome message c5f4ee4 fix: webhook handler null check b7e9d4f chore: update requirements.txt # Visual graph of all branches $ git log --oneline --graph --decorate --all * a3f12c8 (HEAD -> main, origin/main) feat: add /start * c5f4ee4 fix: webhook handler * b7e9d4f chore: update deps # Show commits by a specific author $ git log --author="devtint" # Show commits touching a specific file $ git log --follow bot.py # Show last 5 commits $ git log -5 --oneline # Search commits by message keyword $ git log --grep="webhook"

git diff — what actually changed

terminal
# Unstaged changes (working dir vs. last commit) $ git diff # Staged changes (what's about to be committed) $ git diff --staged # (same as --cached) # Diff between two commits $ git diff c5f4ee4..a3f12c8 # Diff between two branches $ git diff main..feature/webhook # What changed in a specific file only $ git diff main -- bot.py # Just show file names that changed $ git diff --name-only # Show stats (how many lines changed per file) $ git diff --stat bot.py | 15 +++++++++++++++ 1 file changed, 15 insertions(+)

git show & git blame

terminal
# Show full details + diff of a commit $ git show a3f12c8 # Show only the files changed in that commit $ git show a3f12c8 --name-only # Blame — who wrote each line and in which commit $ git blame bot.py a3f12c8 (devtint 2026-04-22 42) async def start(update, context): c5f4ee4 (devtint 2026-04-20 43) await update.message.reply_text("Hello!") # Blame a specific line range $ git blame -L 40,55 bot.py

.gitignore Patterns

Tell git which files to never track. Critical for keeping secrets, build artifacts, and OS junk out of your repo.

Pattern syntax

PatternWhat it ignoresExample
filename.extExact file anywhere.env ignores all .env files
*.extAll files with extension*.pyc ignores all .pyc
folder/Entire directorynode_modules/
/fileOnly in root directory/config.json — root only
**/*.logAny depth**/*.log — logs anywhere
!fileUn-ignore (whitelist)!.env.example — keep this one

Complete template for Python + Node projects

.gitignore
# ── Secrets (NEVER commit these) ───────────────────── .env .env.local .env.production *.pem *.key client_secret*.json service-account*.json !.env.example # ← Un-ignore: commit this as a template # ── Python ─────────────────────────────────────────── __pycache__/ *.pyc *.pyo *.pyd .venv/ venv/ env/ *.egg-info/ dist/ build/ .pytest_cache/ .mypy_cache/ .ruff_cache/ # ── Node / JS ───────────────────────────────────────── node_modules/ .next/ .nuxt/ dist/ .cache/ *.log npm-debug.log* .pnp.* .yarn/cache # ── OS files ────────────────────────────────────────── .DS_Store Thumbs.db desktop.ini # ── Editors ─────────────────────────────────────────── .vscode/settings.json !.vscode/extensions.json # ← keep extension recommendations .idea/ *.swp *.swo # ── Docker / misc ───────────────────────────────────── *.sqlite *.db *.dump uploads/ temp/

Already committed a file that should be ignored?

terminal — remove tracked file from git
# Remove a file from git tracking without deleting it locally $ git rm --cached .env rm '.env' # Remove a whole folder from git tracking $ git rm -r --cached node_modules/ # After removing, commit the change $ git commit -m "chore: remove .env from tracking" # ⚠ If .env contained real secrets, rotate them immediately! # The file still exists in git history — anyone can see it. # Use BFG Repo Cleaner or git filter-repo to purge history.
⚠️

Adding to .gitignore doesn't un-track already committed files. You must run git rm --cached filename to stop tracking it, then commit. The file history will still be visible — rotate any exposed secrets immediately.

Global gitignore (applies to all repos on your machine)

terminal
# Create a global ignore file $ echo ".DS_Store" >> ~/.gitignore_global $ echo "Thumbs.db" >> ~/.gitignore_global $ echo ".idea/" >> ~/.gitignore_global # Tell git to use it $ git config --global core.excludesfile ~/.gitignore_global
