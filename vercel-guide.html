First Deployment

Vercel is the fastest way to go from code to live URL. It supports Next.js, React, Vue, Svelte, Astro, vanilla HTML, and more — zero config needed for most frameworks.

Deploy from GitHub — the standard way

1

Push your code to GitHub

Your project needs to be in a GitHub (or GitLab/Bitbucket) repository. Make sure you have an index.html, package.json, or a framework entry point.

2

Go to vercel.com → Add New Project

Sign in with GitHub. Click Add New → Project. Import your repository from the list.

3

Vercel auto-detects your framework

It reads your package.json and sets the correct Build Command and Output Directory. For Next.js it's automatic — nothing to change.

4

Add environment variables (if needed)

Expand the Environment Variables section before deploying. You can also add them later in Project Settings.

5

Click Deploy

Vercel builds and deploys in ~30–90 seconds. You get a live URL like your-project.vercel.app immediately.

Deploy via CLI (faster for iteration)

terminal
# Install CLI globally npm install -g vercel # Login vercel login # Deploy from project folder vercel # Deploy to production (skips preview) vercel --prod # Pull env vars from Vercel to local .env vercel env pull .env.local

02

Environment Variables

Never hardcode secrets. Vercel encrypts and injects environment variables at build time and runtime. They're available in serverless functions via process.env.

Adding variables

Project Settings → Environment Variables
Go to your project on vercel.com → Settings → Environment Variables. Add key-value pairs for each environment (Production, Preview, Development).
Choose environment scope
Production = main branch deploys only. Preview = all PR/branch deploys. Development = pulled locally via vercel env pull.
Mark secrets as sensitive
Toggle the lock icon to encrypt the value — it won't be readable again after saving. Use this for API keys and tokens.
Redeploy to apply changes
Env var changes don't auto-redeploy. Go to the Deployments tab → ··· → Redeploy on the latest deployment.
⚠️

Next.js prefix rule: Only variables prefixed with NEXT_PUBLIC_ are exposed to the browser. All others are server-only (serverless functions). Use this to your advantage — never expose private keys to the client.

Variable naming

VariableAccessExample use
NEXT_PUBLIC_*Browser + ServerSupabase URL, public API keys
DATABASE_URLServer onlyPostgres connection string
API_SECRET_KEYServer onlyStripe secret, Resend key
NODE_ENVAuto-setproduction on Vercel deploys

Pull env vars locally

terminal
# Pulls all "Development" env vars into .env.local vercel env pull .env.local # .env.local is git-ignored by default in Next.js — never commit it

Serverless Functions

Any file in app/api/ (App Router) or pages/api/ (Pages Router) becomes a serverless function. No server to manage — Vercel spins up containers on demand.

⚠️

Free tier limits: 100 GB-hours compute / month, 10 second max execution time per function on Hobby. For longer jobs (scraping, PDF generation), use Render instead.

API route — App Router (Next.js 14+)

app/api/hello/route.ts
import { NextRequest, NextResponse } from 'next/server' export async function GET(req: NextRequest) { return NextResponse.json({ message: 'Hello from Vercel!' }) } export async function POST(req: NextRequest) { const body = await req.json() // process body.email, body.name etc. return NextResponse.json({ ok: true }, { status: 201 }) }

Edge Runtime (even faster, more limits)

app/api/edge-example/route.ts
export const runtime = 'edge' // runs in Vercel Edge Network globally export async function GET() { return new Response('Ultra-fast edge response', { headers: { 'content-type': 'text/plain' } }) }

04

Custom Domains

Connect your own domain for free. Vercel handles SSL certificate issuance and renewal automatically. No manual cert configuration needed.

Connect a domain

1

Project Settings → Domains → Add

Type your domain (e.g. myapp.com or www.myapp.com). Vercel shows you the DNS records to add.

2

Add DNS records at your registrar

For Cloudflare DNS: Add an A record pointing to Vercel's IP 76.76.21.21, or a CNAME pointing to cname.vercel-dns.com.

3

Wait for propagation

Usually 1–5 minutes with Cloudflare. Vercel checks DNS automatically and issues TLS once verified.

💡

Cloudflare + Vercel tip: Set Cloudflare proxy status to DNS only (grey cloud) on the A/CNAME record pointing to Vercel. If you use Cloudflare's proxy (orange cloud), set SSL mode to Full to avoid redirect loops.

05

CI/CD & Preview Deployments

Every git push triggers a deployment automatically. Vercel creates preview URLs for every branch and PR — your team can review live changes before merging to main.

How it works

🌿

Push to main → Production

Any push to your production branch (usually main) deploys to your live domain instantly.

🔍

Push to any branch → Preview

Feature branches get a unique preview URL like your-project-git-feature-abc.vercel.app. Share it with teammates or clients.

💬

PR Comments

Vercel posts the preview URL as a comment on your GitHub PR automatically. One-click review.

↩️

Instant Rollback

Go to Deployments → click any previous deployment → Promote to Production. Zero-downtime rollback in seconds.

Custom build pipeline (vercel.json)

vercel.json
{ "buildCommand": "npm run build", "outputDirectory": "dist", "installCommand": "npm ci", "rewrites": [ { "source": "/api/(.*)", "destination": "https://my-api.onrender.com/api/$1" } ], "headers": [ { "source": "/api/(.*)", "headers": [ { "key": "Access-Control-Allow-Origin", "value": "*" } ] } ] }
💡

Proxy your Render backend through Vercel rewrites to avoid CORS issues. Requests to /api/* on your Vercel domain get forwarded to Render — the browser never sees a cross-origin request.

Free Tier Limits & Gotchas

The Vercel Hobby plan is surprisingly generous for personal projects. Here's exactly what you get and what to watch out for:

ResourceHobby (Free)Tips
Bandwidth 100 GB / month More than enough for most apps. Monitor in the Usage dashboard.
Serverless Invocations Unlimited Functions are pay-per-use, free tier has generous compute hours.
Function Duration 10 sec max Long-running tasks → use Render. Or split into smaller functions.
Build Time 6,000 min / month Each deploy uses ~1–3 min. 6,000 min = ~3,000 deploys/month.

Common gotchas

⏱️

10-second timeout

Anything that takes longer (image processing, large DB queries) will return a 504. Move heavy work to Render or split into multiple smaller calls.

🚫

No persistent filesystem

Serverless functions are stateless. Writing files to /tmp works within a single invocation but don't rely on it persisting. Use Supabase Storage instead.

🧊

Cold starts

First request after inactivity incurs a ~200ms cold start. Edge runtime reduces this to ~5ms. For latency-sensitive paths, use Edge.

👤

Hobby = personal only

The Hobby plan is for non-commercial personal projects per Vercel's ToS. Commercial projects need Pro ($20/mo per user).

▲

Best alternatives if you hit limits: Cloudflare Pages (unlimited bandwidth, no cold starts), Netlify (similar free tier), or self-host on a $5 VPS with Nginx.
