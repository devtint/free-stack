01

DNS Setup

Move your domain's DNS to Cloudflare. It's the first step — everything else (proxy, Workers, R2) flows from here. Free on all domains.

1

Add your site to Cloudflare

dash.cloudflare.com → Add a Site → enter your domain → Select Free plan.

2

Cloudflare scans existing DNS

It auto-imports your records. Review them — make sure A, CNAME, MX records are correct.

3

Update nameservers at your registrar

Cloudflare gives you two nameservers (e.g. ada.ns.cloudflare.com). Replace your registrar's nameservers with these. Takes 5–48 hours.

4

Verify activation

Cloudflare sends an email when DNS is active. The status changes to Active in the dashboard.

02

Proxy Setup with Vercel & Render

The orange cloud 🟠 = proxied through Cloudflare. The grey cloud ⬜ = DNS only. Understanding which to use where is critical.

🟠

Proxied (orange cloud)

Traffic goes through Cloudflare's CDN. You get DDoS protection, caching, and your origin IP is hidden. Use for most records.

⬜

DNS only (grey cloud)

Direct connection to origin. No Cloudflare features. Required for some services like mail servers.

Cloudflare + Vercel (recommended setup)

1

Set A record to Vercel's IP, proxied 🟠

Name: @, Content: 76.76.21.21, Proxy: ON (orange).

2

Set Cloudflare SSL to "Full"

SSL/TLS → Overview → Set to Full (not Full Strict). This prevents redirect loops between Cloudflare and Vercel.

3

Add domain to Vercel project

Vercel → Project → Settings → Domains → Add your domain.

⚠️

SSL mode must be "Full" or "Full (Strict)" when using Cloudflare proxy with Vercel or Render. If you leave it on "Flexible", you'll get an infinite redirect loop (ERR_TOO_MANY_REDIRECTS).

Cloudflare + Render

1

Add CNAME pointing to your Render URL, DNS only ⬜

Name: api, Content: your-app.onrender.com, Proxy: OFF (grey cloud). Render handles its own SSL.

2

Add the custom domain in Render

Render → Service → Settings → Custom Domains → Add api.yourdomain.com.

04

R2 Object Storage

R2 is Cloudflare's S3-compatible object storage. Free: 10 GB storage, 1M Class A ops, 10M Class B ops/month. Zero egress fees — you can serve files to users without paying for bandwidth.

Create a bucket & upload files

1

Cloudflare Dashboard → R2 → Create bucket

Give it a name. Choose a region (auto is fine for free plan).

2

Create R2 API Token

R2 → Manage R2 API Tokens → Create token. Save the Access Key ID and Secret Access Key — shown only once.

3

Make bucket public (optional)

R2 → your-bucket → Settings → Public Access → Allow. Then you get a public URL like pub-xxx.r2.dev/filename.

Upload from Python

upload.py (boto3 — R2 is S3-compatible)
import boto3 s3 = boto3.client( "s3", endpoint_url=f"https://{ACCOUNT_ID}.r2.cloudflarestorage.com", aws_access_key_id=R2_ACCESS_KEY, aws_secret_access_key=R2_SECRET_KEY, region_name="auto", ) s3.upload_file("local_file.jpg", "my-bucket", "remote/path/file.jpg") # Generate a signed URL (expires in 1 hour) url = s3.generate_presigned_url( "get_object", Params={"Bucket": "my-bucket", "Key": "remote/path/file.jpg"}, ExpiresIn=3600 )
💡

Use R2 instead of local filesystem on Render. Render free tier has no persistent disk. Store uploads, generated files, and assets in R2. It's free, fast, and globally distributed.
